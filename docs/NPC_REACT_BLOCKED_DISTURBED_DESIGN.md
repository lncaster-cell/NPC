# Реакции на blocked/disturbed — краткий диздок

## 1) Цель
Обрабатывать локальные помехи/инциденты без разрушения рутины и без дорогих широковещательных операций.

## 2) Канон
- `OnBlocked`/`OnDisturbed` — это триггеры реакций, а не отдельная «жизнь NPC».
- Реакции ограничены по времени и по числу попыток.
- После реакции NPC либо продолжает шаг, либо уходит в безопасный resync.

## 3) Поведение
- При blocked: попытка локального восстановления движения.
- При disturbed: краткая реакция/переключение состояния и затем возврат к routine.
- Для area-реакций используется bounded fan-out по nearby NPC.

## 4) Ограничения
- Нельзя превращать react-слой в бесконечный retry-loop.
- Нельзя держать NPC в вечном react-state.
- Нельзя заменять routine-систему чисто реактивным поведением.

## 5) Сценарии
- A: временное препятствие -> успешный resume.
- B: серия неудач -> reset runtime и resync.
- C: локальный инцидент группы -> bounded fan-out, затем нормализация.
