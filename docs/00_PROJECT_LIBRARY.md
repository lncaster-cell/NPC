# Ambient Life v2 — Project Library

Дата: 2026-04-02  
Статус: Active (runtime + documentation sync)

## 1) Роль документа
`00_PROJECT_LIBRARY.md` — служебный routing-слой.
Он связывает канон, журнал решений и карту идей без дублирования механик.

## 2) Иерархия чтения
1. **`README.md`** — короткий вход.
2. **`docs/17_UNIFIED_GAME_DESIGN_BRIEF_RU.md`** — единственный high-level канон.
3. **`docs/12_MASTER_PLAN.md`** — супер-краткий индекс.
4. **Доменные SoT**: `12A`, `12B`, `12C`, `12D`, `12E`, `13`, `14`.
5. **Специализированные design-spec**: `20_NPC_BEHAVIOR_SYSTEM_DESIGN_RU.md` (детальная модель поведения NPC).
6. **Служебные документы**: `10` (решения), `16` (sync-map), `18` (режим этапа), `21` (операционный контроль), `architecture/*` (вспомогательные карты).
7. **Инспекционные отчёты документации/кода**: `30_AUDIT_AND_INSPECTION_INDEX.md` (единый вход), где агрегированы `23/24/25/27/28/29` как traceability-снимки.

## 3) Routing-правила
- Меняется legal/world model → `12A`.
- Меняется runtime-концепция → `12B`.
- Меняется property/camp/assets ownership → `12C`.
- Меняется travel → `12D`.
- Меняется trade/city-state → `12E`.
- Меняется clans → `14`.
- Меняется aging/succession → `13`.
- Новое решение/компромисс → `10_DECISIONS_LOG.md`.
- После правок синхронизировать карту идей в `16`.

## 4) Принципы чистоты репозитория
- Не создавать второй «главный обзор» кроме `17`.
- Не раздувать `README` и `12_MASTER_PLAN` до дублирования канона.
- Держать служебные документы явно вторичными к доменным SoT.
- Поддерживать только актуальные ссылки между документами.

## 5) Быстрый чеклист обновления документации
- Сначала обновляйте профильный SoT-документ домена (`12A–12E`, `13`, `14`).
- Если изменены границы ответственности или принципы — добавляйте запись в `10_DECISIONS_LOG.md`.
- Обязательно обновляйте `16_IDEA_INVENTORY_AND_SYNC_MAP.md` после изменения статуса инициативы.
- Проверяйте, что обзорные файлы (`README.md`, `12_MASTER_PLAN.md`, `00_PROJECT_LIBRARY.md`) не дублируют механику из канонов.
- Для команды используйте регламент из `README.md` (раздел «Как обновлять документацию»), как единый Definition of Done.
