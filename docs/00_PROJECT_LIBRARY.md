# Ambient Life v2 — Project Library

Дата: 2026-03-15  
Статус: Active (documentation-first)

## 1) Роль документа
`00_PROJECT_LIBRARY.md` — это навигационный слой.
Он связывает канон, журнал решений и карту идей без дублирования правил.

## 2) Основная структура документации
1. **Канон доменов**: `12A`, `12B`, `12C`, `12D`, `12E`.
2. **Навигация и синхронизация**: `12_MASTER_PLAN`, `16_IDEA_INVENTORY_AND_SYNC_MAP`, `docs/library/DOMAIN_INDEX.md`.
3. **Режим и ограничения этапа**: `18_REBUILD_RESET_CONTEXT.md`.
4. **Архитектурные решения**: `10_DECISIONS_LOG.md`.

## 3) Routing-правила
- Меняется legal/world model → `12A`.
- Меняется runtime-концепция → `12B`.
- Меняется property → `12C`.
- Меняется travel → `12D`.
- Меняется trade/city-state → `12E`.
- Новое решение/компромисс → `10_DECISIONS_LOG.md`.
- После правок синхронизировать карту идей в `16`.

## 4) Принципы чистоты репозитория
- Не держать отдельные временные трекеры, дублирующие канон.
- Удалять устаревшие служебные документы после консолидации.
- Поддерживать только актуальные ссылки между документами.
