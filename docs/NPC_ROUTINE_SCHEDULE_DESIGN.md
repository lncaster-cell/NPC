# Рутины и расписание NPC — краткий диздок

## 1) Цель
Сделать поведение жителей предсказуемым и «живым» через расписание, без heartbeat-поллинга на каждом NPC.

## 2) Канон
- NPC выполняет шаги рутины по событиям и area-тика.
- Выбор активности идёт из контентных locals/waypoints, а не из hardcoded-скриптов «под конкретного NPC».
- Любое прерывание рутины должно иметь безопасный возврат (resume/resync).

## 3) Источник истины
- Контент задаёт маршруты и шаги (`al_route_tag`, `al_step`, связанные waypoint).
- Runtime хранит только состояние исполнения текущего шага/реакции.

## 4) Ограничения
- Нельзя строить отдельный per-NPC heartbeat-loop.
- Нельзя делать world-scan для поиска «чем заняться».
- Нельзя оставлять NPC в зависшем промежуточном состоянии после прерываний.

## 5) Интеграция
- Исполнение идёт через существующие OnSpawn/OnUserDefined/area-centric dispatch.
- После materialization NPC возвращается в общий lifecycle и продолжает штатное расписание.

## 6) Сценарии
- A: штатный шаг рутины завершён -> переход к следующему.
- B: шаг прерван реакцией -> после нормализации выполняется resume.
- C: состояние повреждено/неполное -> выполняется bounded resync.
