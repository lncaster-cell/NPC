# Маршруты и переходы между area — краткий диздок

## 1) Цель
Обеспечить корректную и дешёвую навигацию NPC между точками рутины и связанными area.

## 2) Канон
- Маршруты контент-управляемые: runtime исполняет, но не «изобретает» маршрут на лету.
- Переходы между area идут через связанный граф (`al_link_*`) и штатные transition-хуки.
- Любая операция перехода — bounded и с диагностикой.

## 3) Разделение ответственности
- Контент: правильная разметка route/link.
- Runtime: безопасный move/transition + сохранение консистентного состояния.

## 4) Ограничения
- Запрещены тяжёлые глобальные сканы мира.
- Запрещён «телепорт куда угодно» без route/link-контракта.
- Ошибки разметки не должны приводить к бесконечным циклам/спаму.

## 5) Сценарии
- A: стандартный переход в linked-area -> NPC продолжает рутину.
- B: переход временно недоступен -> deferred/retry в рамках bounded-логики.
- C: неконсистентный route-step -> safe reset + resync.
